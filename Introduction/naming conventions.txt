the naming conventions are used to identify the different entities, 
1)class -> every word must start with upper case letter , if the class Name contains 2 or more words, then start each word with capital letter

example: TypeCasting, ClassRoomInformation, Student....


2) variable/method name -> every variable or method must start with the camel case conventions i,e like the first letter in the first word of method or variable must be lower case ,then after that all the starting letters of the words must be captial letter 

example: getName(),identityModule(),mostFreqentlyRepeatingChar,.....


if we dont follow this conventions , compiler will not throw error or somthing like that but its a good to follow.
